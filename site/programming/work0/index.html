    Работа с Git¶

    Дисциплина: Программирование
    Выполнил: Махов Владислав Анатольевич
    Тема Moodle: Общее — «Работа с Git»

    Задание¶

    Посмотреть скринкаст и повторить показанные действия в командной строке, отразив выполненные шаги в отчёте.

    Выбранный вариант GUI-клиента: Вариант 4 — Встроенные средства IDE (VS Code).

    1. Работа в командной строке (CLI)¶

    В ходе выполнения задания были воспроизведены действия из обучающего скринкаста. Команды выполнялись в терминале Git Bash.

    1.1 Базовая настройка и инициализация¶

    Перед началом работы заданы глобальные параметры пользователя:

    git config --global user.name "Vladislav Makhov"
git config --global user.email "vladscoot39@gmail.com"

    Создан новый каталог и инициализирован пустой репозиторий:

    mkdir my_project
cd my_project
git init

    1.2 Жизненный цикл файлов и фиксация изменений¶

    Создан файл, проверен статус, добавлен в индекс и зафиксирован:

    touch readme.txt
git status               # файл отображается как untracked
git add readme.txt
git commit -m "First commit"

    После изменения содержимого файла проверена разница и зафиксированы изменения:

    git diff                 # показывает изменения
git add readme.txt
git commit -m "Update readme"

    Просмотр истории коммитов:

    git log --oneline

    1.3 Работа с ветками (Branching & Merging)¶

    Создана и выбрана новая ветка для безопасной работы:

    git checkout -b clean_up

    После фиксации изменений в новой ветке выполнено слияние с главной:

    git checkout master
git merge clean_up
git branch -d clean_up   # удаление ветки после слияния

    Просмотр дерева коммитов:

    git log --graph --all --oneline

    1.4 Синхронизация с удалённым репозиторием¶

    git remote add origin https://github.com/Blazeeed/portfolio-prog
git push -u origin master
git pull origin master

    2. Исследование GUI-клиента для работы с Git¶

    Выбранный инструмент: Вариант 4 — Встроенные средства IDE (VS Code).
    Причина выбора: Бесшовная интеграция редактора кода и системы контроля версий — не нужно переключаться между приложениями.

    Выполнение основных операций в VS Code¶

    1. Индексация файлов (git add):
      На вкладке «Source Control» (Ctrl+Shift+G) отображаются изменённые файлы. Нажатие «+» рядом с файлом переносит его в «Staged Changes».

    2. Создание коммита (git commit):
      Ввод сообщения в поле «Message» и нажатие ✓ (Commit). Аналог git commit -m "..." без командной строки.

    3. Синхронизация с удалённым репозиторием (git push / pull):
      Кнопка «Sync Changes» выполняет push и pull в один клик.

    4. Работа с ветками (git branch / checkout):
      Клик на название текущей ветки (левый нижний угол) открывает меню: создать новую ветку, переключиться на существующую, удалить ветку.

    Операция CLI Аналог в VS Code
    git add «+» в Source Control
    git commit -m "..." Поле Message + ✓
    git push «Sync Changes» / «Publish Branch»
    git checkout -b branch «Create new branch...»
    git log --graph Расширение Git Graph
